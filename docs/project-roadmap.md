# Project Roadmap

This document is the public-facing roadmap for the `zenova.sk` website and showcase repository.

It should contain:

- public next steps that are safe to show in GitHub
- work that improves the website, architecture, or delivery story
- items that help explain where the project is going

It should not contain:

- secrets
- internal-only planning
- sensitive commercial notes
- customer-confidential follow-up details

## Why This Exists

For this repository, a roadmap is more useful than scattered TODO comments.

It helps in three ways:

- visitors can see what is already done and what is next
- future GitHub Issues can be opened from clear roadmap items
- the repo feels like a maintained public project, not just a code dump

## Recommended Workflow

- keep this file high-level and public-safe
- use GitHub Issues for individual actionable tasks
- update the roadmap when a milestone changes, not for every tiny edit
- keep deferred but still relevant work in [Public task backlog](./public-task-backlog.md)
